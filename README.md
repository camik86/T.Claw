# T.Claw - 多模态 AI 鼠标键盘控制器

> ⚠️ **Demo 阶段**：本项目目前处于早期开发阶段，功能正在不断完善中。

## 📖 项目简介

T.Claw 是一个基于多模态 AI 的桌面控制助手，通过 AI 视觉识别和智能决策来控制鼠标和键盘操作。项目采用两级截图定位系统，让 AI 能够像人类一样"看到"屏幕并精确操作。

### 核心特性

- 🖱️ **实时鼠标控制**：平滑移动、点击、拖拽、滚轮
- ⌨️ **键盘输入**：文字输入、组合键、单键操作
- 📸 **两级截图定位系统**：
  - **概览图**：全屏 200px 网格，A-Z 纵轴 + 数字横轴标注
  - **详细图**：3x3 块矩阵，中心块放大 4 倍，带详细坐标标注
- 🎯 **精确定位**：每个网格交叉点标注绝对坐标，避免估算误差
- 👁️ **实时掌控**：每次操作前后自动截图确认，模拟人类"动一下看一下"的操作方式
- 🤖 **单步执行模式**：每次只执行一个操作，确认后再规划下一步

## 🚀 快速开始

### 环境要求

- Python 3.8+
- Windows 10/11
- 网络连接（用于调用 AI API）

### 安装依赖

```bash
pip install -r requirements.txt
```

### 配置步骤

1. **启动应用**
   ```bash
   python app.py
   ```

2. **访问设置页面**
   - 打开浏览器访问：`http://localhost:12101/settings`

3. **配置 API 密钥**
   - **API Key**：输入你的火山方舟 ARK API Key
   - **模型选择**：
     - `豆包 2.0 Pro` (推荐，预置接入点)
     - `豆包 2.0 Code Preview` (代码优化)
     - `Doubao-Seed-2.0-Pro`
     - `Doubao-Seed-2.0-Lite`
     - `Doubao-Seed-2.0-Mini`
   - **Endpoint ID**：使用预置接入点时留空即可

4. **保存配置**
   - 点击"保存配置"按钮
   - 系统会自动跳转到聊天页面

### 开始使用

1. **访问聊天页面**
   - 打开浏览器访问：`http://localhost:12101`

2. **输入指令**
   - 在输入框中输入你的需求，例如：
     - "在桌面新建一个文本文档，命名为 hello.txt"
     - "打开记事本"
     - "点击桌面上的 Chrome 图标"

3. **查看执行过程**
   - AI 会自动截图、分析、执行操作
   - 每次操作后都会截图确认结果
   - 在聊天界面查看执行进度

## 📸 两级截图系统详解

### 1. 概览图 (screenshot_overview)

**特点：**
- 全屏截图，每 200 像素一个蓝色网格
- 纵轴：A-Z 字母标注（A=第 0 行，B=第 1 行...）
- 横轴：数字标注（0=第 0 列，1=第 1 列...）
- 每个方块中间显示编号（如 A0、C12）

**用途：** 快速定位目标在屏幕的大致位置

**示例：**
```
目标在 C5 块 = 第 2 行第 5 列 = Y=400, X=1000 附近
```

### 2. 详细图 (screenshot_detail)

**特点：**
- 截取 3x3 块矩阵（共 9 个块），所有块都放大 4 倍
- 中心块：20 像素红色网格，每个交叉点标注绝对坐标（黄色文字）
- 周围 8 个块：只显示块编号，便于感知周围环境

**用途：** 在已知大致位置后，精确定位目标的 exact 坐标

**示例：**
```
screenshot_detail("C5") → 中心块显示 (1050,430) 等精确坐标
```

## 🎯 标准操作流程

AI 遵循的单步执行模式：

1. **截图** → 查看当前屏幕状态
2. **分析** → 决定下一步操作（只规划一步）
3. **执行** → 执行这一个操作
4. **确认** → 截图确认操作结果
5. **决策** → 根据结果决定是否继续

**示例流程：**
```
用户：点击桌面上的 Chrome 图标

AI 执行步骤：
1. screenshot_overview() → 查看全屏
2. 分析：图标在 C5 块
3. screenshot_detail("C5") → 查看详细
4. 分析：图标在 (1050, 430)
5. mouse_move(1050, 430) → 移动鼠标
6. screenshot_overview() → 确认位置
7. 分析：位置正确
8. mouse_click() → 点击
9. screenshot_overview() → 确认结果
```

## 🛠️ 工具列表

### 鼠标操作
- `mouse_move(x, y)` - 移动到指定坐标
- `mouse_click(button, clicks)` - 点击（左/右/中键，单击/双击）
- `mouse_scroll(clicks)` - 滚轮滚动
- `mouse_drag(x, y)` - 拖拽到指定位置

### 键盘操作
- `keyboard_type(text, interval)` - 输入文字
- `keyboard_hotkey(keys)` - 组合键（如 Ctrl+C）
- `keyboard_press(key)` - 单键（如 Enter、Escape）

### 截图操作
- `screenshot_overview()` - 全屏概览图
- `screenshot_detail(block_label)` - 指定块详细图

### 辅助工具
- `get_screen_size()` - 获取屏幕分辨率
- `get_mouse_position()` - 获取鼠标位置

## ⚠️ 注意事项

### Demo 阶段限制

1. **精度限制**
   - 依赖 AI 视觉识别能力
   - 复杂界面可能需要多次尝试
   - 小图标（<20px）识别可能不准确

2. **速度限制**
   - 每次操作都需要截图确认
   - 网络延迟会影响响应速度
   - 建议耐心等待 AI 完成每一步

3. **安全限制**
   - 不会执行危险操作（删除系统文件等）
   - 不会访问敏感区域
   - 建议在人眼监控下使用

### 最佳实践

1. **明确指令**
   - ✅ "点击桌面上的 Chrome 图标"
   - ❌ "打开浏览器"（太模糊）

2. **分步操作**
   - 复杂任务建议分多次对话
   - 每完成一步再指导下一步

3. **耐心等待**
   - AI 需要时间截图、分析、执行
   - 不要频繁打断

## 🔧 开发说明

### 项目结构

```
TempPJ_1/
├── app.py              # Flask Web 应用
├── agent.py            # AI Agent 核心逻辑
├── tools.py            # 工具函数（鼠标、键盘、截图）
├── config.py           # 配置管理
├── templates/          # HTML 模板
│   ├── index.html      # 聊天页面
│   └── settings.html   # 设置页面
├── static/             # 静态资源
│   └── style.css       # 样式表
└── requirements.txt    # Python 依赖
```

### 核心逻辑

1. **Agent 类** (`agent.py`)
   - 管理对话历史
   - 调用 AI API
   - 解析工具调用
   - 执行工具函数

2. **工具系统** (`tools.py`)
   - 所有鼠标键盘操作
   - 截图生成和标注
   - 工具定义和映射

3. **配置系统** (`config.py`)
   - API 密钥管理
   - 模型选择
   - 屏幕参数

### 添加新工具

1. 在 `tools.py` 中实现函数
2. 在 `TOOLS` 列表中添加定义
3. 在 `FUNCTION_MAP` 中添加映射
4. 更新系统提示词
